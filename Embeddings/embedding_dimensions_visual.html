🎯 Understanding Embedding Dimensions

What are Embeddings?

Embeddings convert words, sentences, or any data into numerical vectors (arrays of numbers). The dimension is how many numbers are in that vector.

Think of it like describing a person:
  • 1D: Just height → [170]
  • 2D: Height + weight → [170, 65]
  • 3D: Height + weight + age → [170, 65, 30]
  • 768D: Captures complex semantic meaning!

💡 Key Concepts

1. More dimensions = More information
Higher dimensions can capture more nuanced relationships and meanings.

2. Curse of dimensionality
Too many dimensions can make data sparse and harder to work with.

3. Semantic similarity
Distance between vectors (cosine similarity, Euclidean distance) measures how similar meanings are.

4. Dimensionality reduction
Techniques like PCA, t-SNE, UMAP help visualize high-dimensional data.

🔢 Example: Creating Embeddings

# Simple example with sentence transformers from sentence_transformers import SentenceTransformer model = SentenceTransformer('all-MiniLM-L6-v2') sentences = ["I love machine learning", "AI is fascinating"] embeddings = model.encode(sentences) print(f"Shape: {embeddings.shape}") # (2, 384) # 2 sentences, each represented by 384 dimensions!
